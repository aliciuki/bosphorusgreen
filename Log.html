September 2021

Recent changes include:

  • I fixed file uploading issue. Now, registered users can upload files (images!) directly from the web interface. Yay.
  • I removed Cargo, as it seems to have been interfering with our ability to upload pictures. IIRC, Cargo isn't necessary for any of our extensions.
  • The SemanticTextAnnotator tool was a great find in theory, but the code hasn't been updated and it's probably no longer compatible with our MediaWiki version.
  • I asked for advice at Professional Wiki, where Karsten Hoffmeyer was kind enough to 1) inform me of the above, and 2) point to some good support resources we can go to in order to figure out issues in the future.

June 2021

Hey! This has been on my mind for a long time, and I've just decided to go for it and ask questions later. The best spot for our project data--texts and links between them, including toponymy and geo-references--is a Wiki. This is why I've installed this instance of MediaWiki on a Digital Ocean server. I think this is the way to go.

I'll be able to explain it better as we go along, but apart from the obvious, well-known Wiki functionalities (collaboration, hyperlinking, encyclopedia-like structure), there are two aspects of this software that make it a good choice:

  • Linked Open Data. A Wiki is at its most fundamental a database. So, for example, this wiki software (MediaWiki and Semantic MediaWiki). This means that we can use this platform to 1) continue our tagging efforts and 2) build upon them to create spatial links, and any other classifications that will be needed by our project from now on.
  • APIs If we build our knowledge base here, we can share the resulting structured data via an API, which is great if we want to try multiple visualization tools without necessarily migrating or exporting our data every time. If we do need to export it, the JSON format is a well-established standard, and the data contained in JSON can also be turned into spreadsheets (if needed).

I'm taking care of setting up the wiki to do what we want it to do, and hope to have something flashy to show by Wednesday. I know that we need to show something by August. I've no doubt we'll have an amazing and impressive performance in the conference.

Btw, do you like the logo? It's Godfrey's banana tree, recoloured with Adobe Illustrator. We can make it better, or pick a better logo, but I'm liking the way this is starting to look.
